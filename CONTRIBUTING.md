# Contributing to APN

APMN is an open community standard. All contributions are welcome.

## Ways to contribute

- **Spec clarifications** — open an issue if wording is ambiguous
- **New node type proposals** — open an issue with use case, attributes, and compiler mapping
- **Example workflows** — add to `examples/` via PR
- **Tooling** — BPMN.io plugin, VS Code extension, validators
- **Reference implementations** — in any language (JS, Java, Go, etc.)

## Process

1. Open an issue to discuss before large changes
2. Fork the repo, create a branch (`feature/my-proposal`)
3. Submit a PR with a clear description of the change and rationale
4. Maintainers review within 7 days

## Governance

APMN is currently maintained by [Kshetra Studio](https://kshetra.studio).  
The goal is to move to a community working group model at v1.0.
